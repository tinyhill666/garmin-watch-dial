# fr955 表盘设计能力简报

给表盘视觉设计用的设备能力与约束说明。设计必须完全落在这些能力范围内，否则无法实现。

## 屏幕

- **260 × 260 像素，正圆形**（四角不可用，内容须落在半径 130 的圆内；越靠近边缘可用宽度越窄）
- **MIP 半透反射彩屏，常亮**：不是 AMOLED。特点是阳光下清晰、室内偏暗淡
- **仅 64 色（RGB222）**：每个颜色通道只有 4 级 `0x00 / 0x55 / 0xAA / 0xFF`。
  可用色即形如 `#00AAFF`、`#FF5500`、`#AAAAAA` 这类组合，任意其他色值会被就近量化
- **无透明度混合、无渐变**。抗锯齿可用（圆弧/斜线边缘平滑）
- 深色（纯黑 `#000000`）背景是 MIP 表盘的主流选择，省电且对比度好；浅色背景也可行

## 字体（内置，不可旋转）

| 字体 | 大致高度 | 内容 |
|---|---|---|
| FONT_XTINY | ~22px | 全字符 |
| FONT_TINY | ~26px | 全字符 |
| FONT_SMALL | ~30px | 全字符 |
| FONT_MEDIUM | ~33px | 全字符 |
| FONT_LARGE | ~39px | 全字符 |
| FONT_NUMBER_MILD | ~48px | **仅数字和 : - **|
| FONT_NUMBER_MEDIUM | ~66px | 仅数字和 : - |
| FONT_NUMBER_HOT | ~76px | 仅数字和 : - |
| FONT_NUMBER_THAI_HOT | ~110px | 仅数字和 : - |

- 文本对齐：左/中/右 + 垂直居中，可组合
- 后续可以做自定义位图字体（需出 PNG 字模），v2 先用内置字体

## 绘图原语

- 直线（可设线宽）、矩形、圆角矩形、圆、椭圆、多边形（任意填充多边形）
- **圆弧 `drawArc`**：可设线宽、起止角度、顺/逆时针——做进度环/刻度的核心手段
- 剪裁区域 setClip
- PNG 位图资源（预先做好放进资源目录，可用于图标）
- 没有：文字旋转、渐变、阴影、模糊

## 刷新模型（关键约束）

- **整屏重绘每分钟一次**（低功耗常亮状态）
- **每秒刷新只允许重绘一个小矩形区域**（onPartialUpdate + 剪裁，功耗预算 ~30ms）
  - 秒的呈现只能是"小区域数字"或"小范围图形"，**不能做整圈秒针/秒进度环**（每秒重绘区域太大）
- 抬腕后短时间进入高功耗模式，此时整屏每秒重绘（可以有更活跃的效果，但设计以低功耗态为准）

## 必须展示的数据（本次需求）

- 时（HH）、分（MM）、秒（SS，注意上述小区域约束）
- 心率（bpm，可能暂时无值，需设计 "--" 占位形态）
- 电量（0-100%）
- 步数（0 ~ 99999，注意位数变化时的布局稳定性）
- 周几（SUN..SAT 或图形化）
- 日期：月 + 日

## 可选数据（设计需要时可加）

步数目标进度、卡路里、距离、爬楼、身体电量、压力、天气温度、通知数、蓝牙连接状态、日出日落、闹钟

## 交付格式要求

设计稿要给出：每个元素的**精确坐标/尺寸/字体/颜色（hex）**，坐标系原点在左上角，(130,130) 为圆心。颜色必须是 RGB222 合法值。
