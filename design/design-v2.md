# 表盘视觉方案 v2 —— «AXIS»

> 目标设备 fr955（260×260 正圆，MIP，RGB222 64 色）。坐标系：原点左上，(130,130) 为圆心。
> 所有颜色均为 RGB222 合法值（每通道 ∈ {0x00, 0x55, 0xAA, 0xFF}）。

## 设计理念

整个表盘由一条看不见的**中轴线（x=130）**组织：顶部日期在轴两侧分列、时间压轴居中、数据行用两枚图标夹轴对称、底部步数进度弧沿表缘展开呼应圆形。时分是唯一的大字主角，其余信息按"数据行"统一字号收编为配角；点缀色只有两个且各司其职——活力蓝负责"日历与活动"，橙红只属于"心率与警示"。

## 调色板（6 色，全部 RGB222 合法）

| 用途 | Hex | 出现位置 |
|---|---|---|
| 背景 | `#000000` | 全屏 |
| 主文字 | `#FFFFFF` | 时间、日期数字、心率、电量、步数 |
| 次级文字 | `#AAAAAA` | 秒、电池图标描边、心率无值占位 |
| 结构灰 | `#555555` | 进度弧轨道、顶部分隔点 |
| 活力蓝（点缀 1） | `#00AAFF` | 周几、步数进度弧（上下呼应） |
| 橙红（点缀 2） | `#FF5500` | 心形图标、低电量警示（≤20%） |

## 布局系统

四个水平带 + 一条贝塞尔弧，全部锚定中轴 x=130：

```
y≈40   ┄ 顶带：  WED · 7/2        （轴两侧分列，蓝色周几）
y≈118  ┄ 主带：  10:09  ³⁶        （大字居中，秒小字缀于右下）
y≈180  ┄ 数据行：68% [▯‣ | ♥] 128 （图标夹轴，数值向外生长）
y≈214  ┄ 底带：  8456             （步数居中）
r=118  ┄ 贝塞尔弧：220°→320°       （步数目标进度，蓝）
```

视觉层级：时间 ≫ 数据行/步数 > 日期 > 秒。

## 元素规格表

字体高度按能力简报取值；落地时用 `dc.getFontHeight()` / `getTextWidthInPixels()` 校准，允许 ±2px 微调。文本锚点均使用 Garmin `TEXT_JUSTIFY_*` 常量 + `TEXT_JUSTIFY_VCENTER`。

### 1. 背景

| 属性 | 值 |
|---|---|
| 绘制 | 全屏填充 `#000000` |

### 2. 顶带 —— 周几 + 日期（y=40）

| 元素 | 内容 | 字体 | 颜色 | 对齐 | 锚点 (x,y) |
|---|---|---|---|---|---|
| 周几 | `WED`（SUN..SAT，3 字母大写） | FONT_TINY (~26px) | `#00AAFF` | RIGHT \| VCENTER | (124, 40) |
| 分隔点 | 实心圆 r=2 | — | `#555555` | — | 圆心 (130, 41) |
| 日期 | `7/2`（M/D，无前导零） | FONT_TINY | `#FFFFFF` | LEFT \| VCENTER | (136, 40) |

- 周几向左生长（固定 3 字母，宽约 45px，落在 x≈79–124）；日期向右生长（最宽 `12/31` ≈ 64px，落在 x≈136–200）。圆边界在此行为 x∈[51,209]，均安全。
- 轴两侧各留 6px 空隙，中央分隔点声明"这是刻意的对轴分列"。

### 3. 主时间 HH:MM（绝对主角）

| 属性 | 值 |
|---|---|
| 内容 | `10:09`（小时补零，宽度稳定） |
| 字体 | FONT_NUMBER_MEDIUM (~66px) |
| 颜色 | `#FFFFFF` |
| 对齐 | CENTER \| VCENTER |
| 锚点 | (130, 118) |
| 占位包络 | 约 168×66，x≈46–214，y≈85–151 |

### 4. 秒（缀于时间右下，小区域每秒刷新）

| 属性 | 值 |
|---|---|
| 内容 | `36`（两位补零） |
| 字体 | FONT_TINY (~26px) |
| 颜色 | `#AAAAAA` |
| 对齐 | LEFT \| VCENTER |
| 锚点 | (222, 138)（底边与时间数字底缘对齐，x=222 与时间右缘留 ~8px） |

**秒刷新剪裁矩形：`x=220, y=124, w=34, h=28`**（953px²，远低于 onPartialUpdate 预算）。
`onPartialUpdate` 流程：`setClip(220,124,34,28)` → 填 `#000000` → 重绘秒字。矩形最右 x=254，在该行圆边界 (≈257) 内。

### 5. 数据行 —— 电量（左）+ 心率（右）（y=180，行高 30，y≈165–195）

两枚图标对称夹住中轴（内缘 x=125 / x=136），数值分别向左右外侧生长，位数变化不影响轴心结构。

| 元素 | 内容 | 字体 | 颜色 | 对齐 | 锚点/几何 |
|---|---|---|---|---|---|
| 电量数字 | `68%`（0–100%） | FONT_SMALL (~30px) | `#FFFFFF`；≤20% 时 `#FF5500` | RIGHT \| VCENTER | (98, 180) |
| 电池图标·外框 | 圆角矩形描边，pen=2，r=3 | — | `#AAAAAA` | — | (104, 173, w18, h14) |
| 电池图标·电量条 | 实心矩形，宽 = round(12 × 电量%) | — | `#FFFFFF`；≤20% 时 `#FF5500` | — | (107, 176, w0–12, h8) |
| 电池图标·触点 | 实心矩形 | — | `#AAAAAA` | — | (122, 177, w3, h6) |
| 心形图标 | 2 圆 + 三角（见下方图标细则） | — | `#FF5500` | — | 包络 (136–152, 173–187) |
| 心率数字 | `128`（bpm） | FONT_SMALL | `#FFFFFF` | LEFT \| VCENTER | (158, 180) |

图标绘制细则（全部原语，无需位图资源）：
- **心形**：`fillCircle(140,177,4)` + `fillCircle(148,177,4)` + `fillPolygon [(136,178),(152,178),(144,187)]`，成品 16×14。
- **电池**：`setPenWidth(2)` + `drawRoundedRectangle(104,173,18,14,3)`；电量条与触点用 `fillRectangle`。电池图标本身就是模拟量显示，数字提供精度。

### 7. 步数目标进度弧（签名元素，沿表缘）

| 属性 | 值 |
|---|---|
| 圆心 / 半径 | (130,130) / r=118（弧中心线），pen=8 → 覆盖 r114–122，距表缘留 8px |
| 角度系 | Garmin 约定：0°=3 点钟方向，角度逆时针增大 |
| 轨道 | `drawArc(130,130,118, ARC_COUNTER_CLOCKWISE, 220, 320)`，`#555555` |
| 进度 | `drawArc(130,130,118, ARC_COUNTER_CLOCKWISE, 220, 220 + 100×min(steps/goal, 1))`，`#00AAFF`；goal 取用户步数目标（默认 10000）；steps=0 时不画进度段 |
| 屏幕端点参考 | 左端 (39.6, 205.9)、底部中点 (130, 248)、右端 (220.4, 205.9)；示例 84.6% 时进度终点 (197.0, 227.2) |

进度从左端起沿底部向右生长（视觉逆时针），达标后封满 100° 不再溢出。

## 动态状态与边界情况

| 场景 | 表现 |
|---|---|
| 心率无值 | 数字画 `--`，颜色降为 `#AAAAAA`；心形图标保持 `#FF5500` 不变（身份锚点） |
| 心率 2→3 位 | 左对齐 x=158 起向右生长，最长 `188` 右缘 ≈209，圆边界 ≈242，安全 |
| 电量 100% | `100%` 右对齐止于 x=98，左缘 ≈29，圆边界 ≈17，安全 |
| 电量 ≤20% | 数字与电量条同时转 `#FF5500`，外框保持灰色 |
| 步数 0–99999 | 居中对齐，宽度对称伸缩；`99999` 宽 ≈85px（x≈88–172），与两侧无碰撞 |
| 步数 ≥ 目标 | 进度弧封满 220°–320°，数字继续累加 |
| 抬腕高功耗态 | 布局不变（每秒整屏重绘时秒照常更新，无额外动效，保持一致性） |

## 绘制顺序（onUpdate）

1. 背景黑 → 2. 弧轨道 → 3. 弧进度 → 4. 顶带（周几/点/日期）→ 5. 主时间 → 6. 秒 → 7. 电池图标 + 电量数字 → 8. 心形 + 心率 → 9. 步数。
所有字符串与坐标常量在类成员中预分配，`onUpdate` 内零对象分配。

## 实现备注

- 本表坐标基于能力简报的字体高度估算；接入真机字体后以 `getFontHeight`/`getTextWidthInPixels` 实测，垂直锚点允许 ±2px 微调，**轴对称关系（124/136、98/158、125/136 夹轴）必须保持**。
- 秒使用 FONT_TINY 而非 NUMBER 字体：区域小、功耗低，且灰色弱化不与主时间争夺注意力。

## 风格变体方向（备选，一行一个）

- 主时间换 FONT_NUMBER_HOT（~76px）更霸气：需把秒移入数据行中缝 (130,180) 并接受更满的构图。
- 点缀蓝可整体替换为荧光绿 `#00FF55` 或琥珀 `#FFAA00`（均 RGB222 合法），一次换色即换气质。
- 浅色版：背景 `#FFFFFF`、文字 `#000000`、轨道 `#AAAAAA`，点缀色不变，适合偏好白盘的用户。
